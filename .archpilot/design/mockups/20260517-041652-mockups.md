# Wireframes and Mockups Package

## Document Purpose

This document provides wireframe specifications, user flow diagrams, and mockup guidance for the Product Catalog Management System admin panel. It translates the Product Surface Plan into visual design artifacts to validate UX workflows, screen layouts, and interaction patterns before development.

## Screen Inventory

## Low-Fidelity Wireframe Descriptions

### S01: Login Screen

**Layout:**
- Centered card (400px width) on a neutral background
- Logo at top of card
- Email input field (full width)
- Password input field (full width)
- "Sign In" button (full width, primary color)
- "Forgot Password?" link (right-aligned, below button)

**States:**
- **Default**: Empty form fields, enabled "Sign In" button
- **Loading**: "Sign In" button shows spinner, disabled state
- **Error**: Inline error message below email/password field (e.g., "Invalid email or password")
- **MFA Challenge**: Modal overlay with TOTP code input field, "Verify" button

**Accessibility:**
- Email and password fields have `<label>` elements with `for` attributes
- Error messages announced via `aria-live="polite"`
- Focus trap in MFA modal
- Keyboard navigation: Tab to navigate fields, Enter to submit

**Responsive Notes:**
- Desktop (1920x1080): Centered card, 400px width
- Tablet (1024x768): Centered card, 400px width (same as desktop)

### S02: Dashboard

**Layout:**
- **Top Navigation Bar** (full width, fixed):
  - Logo (left)
  - User menu (right): Avatar, name, dropdown (Profile, Settings, Logout)
- **Sidebar Navigation** (left, 240px width, fixed):
  - "Products" (active)
  - "Categories"
  - "Audit Logs"
  - "Settings"
- **Main Content Area** (right, remaining width):
  - **Quick Stats Row** (top):
    - Card: "Total Products" (count)

    - Card: "Recent Activity" (last 5 audit log entries)

    - Search bar (left)
    - Filter dropdowns (Category, Status, Price Range)
    - "Add Product" button (right)

**States:**
- **Loading**: Skeleton screens for stats cards and product table
- **Empty**: "No products found" message with "Add Product" button

**Accessibility:**
- Sidebar navigation uses `<nav>` with `aria-label="Main navigation"`
- Product table uses `<table>` with `<thead>`, `<tbody>`, `<th scope="col">`

## Responsive Layout Notes

### Breakpoints
- **Desktop**: 1920x1080 (primary target)
- **Tablet**: 1024x768 (secondary target)
- **Minimum Supported Width**: 768px (iPad portrait)

### Layout Strategies
- **Desktop (1920x1080)**:
  - Sidebar navigation: Fixed, 240px width
  - Main content area: Remaining width (1680px)
  - Two-column forms: Left column 60%, right column 40%
  - Product table: All columns visible
- **Tablet (1024x768)**:
  - Sidebar navigation: Collapses to hamburger menu icon
  - Main content area: Full width (1024px)
  - Single-column forms: Full width
  - Product table: Reduced columns (Image, Name, Price, Status, Actions)
  - Touch-friendly controls: Larger hit areas (min 44px x 44px)

### Touch Optimization (Tablet)
- Drag-and-drop handles: Larger hit area (48px x 48px)
- Buttons: Minimum 44px height
- Checkboxes: Larger hit area (32px x 32px)
- Dropdown menus: Touch-friendly (larger padding, larger font size)

## Component States

### Buttons
- **Default**: Enabled, primary or secondary color
- **Hover**: Slightly darker shade (desktop only)
- **Active**: Pressed state (darker shade, slight scale down)
- **Disabled**: Grayed out, cursor not-allowed
- **Loading**: Spinner icon, disabled state

### Form Inputs
- **Default**: Empty, enabled, border color neutral
- **Focus**: Border color primary, focus ring (2px outline)
- **Filled**: Value entered, border color neutral
- **Error**: Border color red, error message below field
- **Disabled**: Grayed out, cursor not-allowed

### Checkboxes
- **Unchecked**: Empty box, border color neutral
- **Checked**: Checkmark icon, background color primary
- **Indeterminate**: Dash icon, background color primary (for "select all" when some selected)
- **Disabled**: Grayed out, cursor not-allowed

### Dropdowns
- **Closed**: Selected value displayed, chevron down icon
- **Open**: Dropdown menu visible, chevron up icon
- **Hover (item)**: Highlighted background (desktop only)
- **Selected (item)**: Checkmark icon, bold text

### Modals
- **Closed**: Not visible
- **Open**: Overlay visible, modal centered, focus trapped
- **Loading**: Spinner in modal body, buttons disabled
- **Success**: Success message, "Close" button
- **Error**: Error message, "Retry" and "Close" buttons

### Tables
- **Default**: Rows visible, no selection
- **Hover (row)**: Highlighted background (desktop only)
- **Selected (row)**: Highlighted background, checkbox checked
- **Sortable (column)**: Chevron icon (up/down), clickable header
- **Expandable (row)**: Chevron icon (right/down), clickable row

### Toasts
- **Success**: Green background, checkmark icon, auto-dismiss after 5 seconds
- **Error**: Red background, error icon, manual dismiss or auto-dismiss after 10 seconds
- **Info**: Blue background, info icon, auto-dismiss after 5 seconds
- **Warning**: Yellow background, warning icon, auto-dismiss after 7 seconds

## Accessibility Notes

### Keyboard Navigation
- **Tab**: Navigate forward through interactive elements
- **Shift + Tab**: Navigate backward through interactive elements
- **Enter**: Activate button, submit form, expand/collapse tree node
- **Space**: Toggle checkbox, activate button
- **Arrow Keys**: Navigate tree view, navigate dropdown menu items
- **Escape**: Close modal, close dropdown menu

### Screen Reader Support
- All form fields have `<label>` elements with `for` attributes
- Error messages announced via `aria-live="polite"`
- Success messages announced via `aria-live="polite"`
- Modals have `role="dialog"`, `aria-labelledby`, `aria-modal="true"`
- Tables have `<caption>`, `<th scope="col">`, `<th scope="row">`
- Tree view uses `role="tree"`, `role="treeitem"`, `aria-expanded`
- Buttons have `aria-label` when icon-only

### Focus Management
- Focus trap in modals: Tab cycles through modal elements only
- Focus returns to trigger element when modal closes
- Focus visible indicator: 2px outline, primary color
- Skip to main content link (hidden, visible on focus)

### Color Contrast
- Text on background: Minimum 4.5:1 contrast ratio (WCAG AA)
- Large text (18pt+) on background: Minimum 3:1 contrast ratio (WCAG AA)
- Interactive elements: Minimum 3:1 contrast ratio for borders and icons
- Error messages: Red text with sufficient contrast, supplemented with icon

### ARIA Attributes
- `aria-label`: Descriptive label for icon-only buttons, drag handles, checkboxes
- `aria-labelledby`: Reference to modal title, form section heading
- `aria-describedby`: Reference to error message, help text
- `aria-live="polite"`: Announce dynamic content updates (toasts, validation errors)
- `aria-expanded`: Indicate expandable/collapsible state (tree nodes, dropdowns)
- `aria-selected`: Indicate selected state (tabs, dropdown items)
- `aria-required`: Indicate required form fields
- `aria-invalid`: Indicate invalid form fields
